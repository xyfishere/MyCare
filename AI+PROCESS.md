# AI and Process

## Role of AI

AI is used to explore options, inspect the codebase, implement scoped changes, test behavior, and document the reasoning behind the work.

The project owner sets product direction and decides what is released.

## Effective Workflow

1. Start with the user problem, not the feature.
2. Review the existing product before proposing a change.
3. Compare a small number of design or implementation options when direction is unclear.
4. Implement one bounded change at a time.
5. Verify the result in the browser and test affected data behavior.
6. Keep local changes separate from publishing.

## Quality Checks

- Verify AI suggestions against the code, documentation, and actual UI.
- Treat generated text, translations, and technical claims as drafts.
- Check privacy, licensing, accessibility, responsive layout, and data migration.
- Prefer proven libraries and official documentation for security-sensitive integrations.

## Known Limits

AI can misunderstand visual intent, overlook edge cases, or produce confident but incorrect information. Human review remains necessary, especially for authentication, permissions, health data, legal claims, and deployment.

## Learning Record

When a useful lesson appears, record:

- What was attempted.
- What worked or failed.
- What was verified.
- What should be repeated or avoided next time.
